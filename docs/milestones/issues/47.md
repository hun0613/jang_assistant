| 브랜치 | `deploy/#47` |

## 문제 분석

### 증상
- 로컬 개발 환경: 데이터 fetch 빠름
- Vercel 배포 환경: 모든 데이터 fetch 요청이 일정하게 느림
- Cold Start가 아님 (첫 요청만 느린 게 아니라 매번 느림)

### 원인
| 항목 | 리전 | 위치 |
|------|------|------|
| Vercel Function | `iad1` | 미국 버지니아 북부 |
| Supabase DB | `ap-northeast-2` | 서울 |

- 두 리전 간 물리적 거리 약 11,000km
- 매 Supabase 쿼리마다 태평양 왕복 RTT(~200ms) 발생
- `Promise.all` 병렬 처리로도 전체 latency가 RTT에 종속됨

### 로컬이 빠른 이유
- 로컬 dev: 본인 PC(한국) → Supabase 서울 → RTT 약 5~20ms
- 배포: 브라우저 → Vercel iad1(미국) → Supabase 서울 → 매번 태평양 왕복

## 작업 로드맵

### Step 1: vercel.json 생성 ✅
- `regions: ["icn1"]` 설정으로 Function Region을 서울로 변경
- Vercel Hobby 플랜에서도 1개 리전 선택 가능

### Step 2: 배포 후 효과 검증
- 재배포 후 Vercel Function Region이 icn1로 변경되었는지 확인
- 실제 데이터 fetch 속도 개선 체감 확인

## 예상 효과
- RTT: ~200ms → 20ms 이하 (약 90% 감소)
- 모든 SSR 페이지의 데이터 로딩 속도 개선
