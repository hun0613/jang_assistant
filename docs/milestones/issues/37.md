# #37 프론트엔드 localStorage → Supabase 마이그레이션

## 이슈 정보

| 항목 | 내용 |
|------|------|
| 마일스톤 | Phase 1 - 백엔드 도입 및 데이터 영속화 |
| 목표 | 기존 localStorage 기반 데이터 로직을 Supabase Server Actions 호출로 전환 |
| 브랜치 | `feat/#37` |
| 참고 | #35(장바구니 CRUD), #36(품목 CRUD) Server Actions 활용. UI 가이드 관련 localStorage는 유지 |

## 스코프 정의

### 마이그레이션 대상 (비즈니스 데이터)
| localStorage 키 | 전환 대상 | 설명 |
|-----------------|----------|------|
| `title` | `carts.title` | 장바구니 제목 |
| `memo` | `carts.memo` | 장바구니 메모 |ㅍ
| `cartItems` | `cart_items` 테이블 | 품목 목록 (생성 시) |
| `shoppingHistory` | `cart_items` 테이블 (status/price) | 쇼핑 중 품목 상태 |

### 마이그레이션 제외 (UI 상태 → localStorage 유지)
| localStorage 키 | 사유 |
|-----------------|------|
| `createCartGuideShown` | 가이드 노출 여부 (UI 상태) |
| `shoppingUsageGuideShown` | 가이드 노출 여부 (UI 상태) |

### 변경 대상 컴포넌트

**`/createCart` 흐름 (4개)**
| 컴포넌트 | 현재 동작 | 전환 후 동작 |
|----------|----------|-------------|
| `CreateCartFormComp` | localStorage에서 title/memo/cartItems 로드, blur 시 저장 | 세션/장바구니 생성 → Supabase, 품목은 DB에서 조회 |
| `AddCartItemModalComp` | cartItems 배열에 push 후 localStorage 저장 | `createCartItem` Server Action 호출 |
| `CreateCartItemListComp` | cartItems 배열에서 splice 후 localStorage 저장 | `deleteCartItem` Server Action 호출 |
| `ResumeShoppingModalComp` | shoppingHistory localStorage 삭제 | 장바구니 상태 업데이트 (SHOPPING → CREATED 등) |

**`/shopping/[id]` 흐름 (3개)**
| 컴포넌트 | 현재 동작 | 전환 후 동작 |
|----------|----------|-------------|
| `ShoppingComp` | localStorage에서 shoppingHistory/cartItems 로드, 변경 시 저장 | `getCartItemsByCartId`로 로드, `updateCartItem`으로 수정 |
| `CompleteShoppingGuideModalComp` | title/memo/cartItems/shoppingHistory 모두 삭제 | 장바구니 상태를 COMPLETED로 업데이트 |
| `NoDataHandlerComp` | cartItems 배열 길이 체크 | Supabase에서 품목 존재 여부 확인 |

## 작업 단계

### Step 1. `/createCart` 흐름 마이그레이션
- sessionUtil 유틸 생성 (세션 생성/복원)
- CreateCartFormComp: 세션/장바구니 생성 로직 Supabase 전환
- AddCartItemModalComp: `createCartItem` 호출로 전환
- CreateCartItemListComp: `deleteCartItem` 호출로 전환
- ResumeShoppingModalComp: 장바구니 상태 업데이트로 전환
- StartShoppingGuideModalComp: 장바구니 SHOPPING 상태 전환 + cartId 라우팅
- 상태: ✅ 완료

### Step 2. `/shopping/[id]` 흐름 마이그레이션
- ShoppingComp: cartId 기반 품목 로드, Supabase 기반 동기화
- PickItemModalComp / DropItemModalComp: `updateCartItem` 호출로 전환
- ShoppingItemListComp: cartId prop 추가
- CompleteShoppingGuideModalComp: 장바구니 COMPLETED 상태 업데이트
- NoDataHandlerComp: Supabase cart 존재 확인으로 전환
- shopping/[id]/page.tsx: DB에서 title 조회하여 metadata 생성
- 상태: ✅ 완료

### Step 3. 불필요한 localStorage 코드 정리
- title/memo/cartItems/shoppingHistory localStorage 호출 제거
- 새 localStorage 키 도입: sessionId, draftCartId, shoppingCartId
- 가이드 관련 키(createCartGuideShown, shoppingUsageGuideShown) 유지
- 상태: ✅ 완료

### Step 4. 빌드 검증
- `pnpm build` 정상 완료 확인
- 상태: ✅ 완료
