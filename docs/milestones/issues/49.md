# #49 preferredRegion='icn1' 설정 롤백

## 이슈 정보

| 항목 | 내용 |
|------|------|
| 마일스톤 | Phase 1.5 - UX 보완 (로딩/에러 처리) |
| 목표 | Hobby 플랜에서 사용 불가능한 preferredRegion 설정 제거 |
| 브랜치 | `bugFix/#49` |

## 배경

- #47에서 `src/app/layout.tsx`에 `export const preferredRegion = 'icn1'` 추가
- Vercel Hobby 플랜에서는 `icn1` (Seoul) 리전이 Pro 전용으로 제한됨
- 현재 설정이 무시되어 실제 Function은 기본값(iad1)에서 실행 중
- 무효한 dead code이므로 제거

## 학습 기록 (스터디 발표용)

### 시도한 방법과 실패 과정

1. **vercel.json의 `regions: ["icn1"]`** → Next.js App Router에서는 적용되지 않음 (Pages Router 레거시 함수 전용)
2. **`export const preferredRegion = 'icn1'`** → App Router에서 올바른 문법이나, Vercel Hobby 플랜에서 icn1은 Pro 전용 리전이라 사용 불가
3. **Vercel Dashboard에서 수동 변경** → 마찬가지로 Hobby 플랜 제약으로 icn1 선택 불가

### 결론
- 근본 원인 (Vercel iad1 ↔ Supabase ap-northeast-2 리전 불일치)은 확인했으나, Vercel 측 리전 변경은 Hobby 플랜에서 불가
- → #50에서 Supabase를 us-east-1로 이전하는 방향으로 피벗
