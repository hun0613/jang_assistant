## 2. 배경 및 기획 의도

### 2.1 문제 정의
오프라인 마트에서 장을 볼 때 많은 사람들이 메모장이나 종이 리스트에 의존하고 있으며, 다음과 같은 불편함을 겪고 있다:

- 현재까지 담은 물품의 **총 금액 파악이 어려움** (ex. 사은품 기준 금액 달성 여부)
- 리스트에 있지만 **카트에 담지 않은 물품을 놓침**
- **과거 구매 이력 확인이 어려움** (가격, 품목 등)
- **상황별 장보기 리스트 관리가 안 됨** (캠핑, 명절 등)

### 2.2 시장 조사
- **경쟁 환경**: 현존하는 장보기 관련 앱(마켓컬리, SSG, 이마트 등)은 온라인 커머스 또는 오프라인 픽업 서비스만 존재. 오프라인 장보기 전용 앱은 부재
- **설문조사** (2025.05.24 ~ 06.08, 구글 폼):
  - 85% 이상이 최소 월 1회 이상 오프라인 마트를 방문 (주 1회 이상 55%, 월 1회 이상 30%)
  - 55.5% 이상이 사야할 품목을 기록·체크하며 장을 봄 (스마트폰 메모 44.4%, 종이 메모 11.1%)
  - 사용자 의견: 가격 예측, 매장별 할인정보, 매장 간 가격비교, 편한 UI의 전용 앱 니즈

### 2.3 Pain Point & Solution

| Pain Point | Solution |
|------------|----------|
| 마트에서 물건을 담고 나서야 얼마 나왔는지 알 수 있어서 지출관리가 어려움. 충동구매도 자주 발생 | **실시간 금액 합산으로 예산 초과 방지** - 카트에 담긴 물품의 총액을 자동 계산하여 실시간으로 지출 예측 |
| 메모장이랑 계산기랑 번갈아 쓰는 과정이 불편하고 글씨도 작아서 조작 실수도 많이 발생 | **올인원 장보기 관리 기능** - 장보기 리스트와 실시간 합산 기능을 한 곳에 배치. 크고 보기 쉬운 UI |
| 지난번에 뭘 샀는지, 얼마 썼는지 기억이 안 나고 같은 요리를 하려면 또 재료를 검색해야 해서 번거로움 | **구매 히스토리와 금액 저장 기능** - 과거 장보기 내역을 저장하고 테마별 물품 리스트와 품목별 가격 및 총 지출 이력 확인 |

### 2.4 목표 사용자

**메인 타겟: #눈으로 직접 고르는 계획형 소비자**
- 직접 눈으로 보고 만져보며 품질을 확인하고, 필요한 것들을 미리 리스트로 정리하여 관리하는 계획형 소비자
- 오프라인 마트에서 정기적으로 장을 보는 사람
- 스마트폰으로 쇼핑 리스트를 관리하고 싶은 사람
- 지출을 체계적으로 관리하고 싶은 사람

| Needs | 카트에 담으면 금액이 자동 합산되는 기능, 한 손으로 쉽게 물품을 체크하고 입력할 수 있는 간단하고 큼직한 UI | 과거에 뭘 샀고 얼마 나왔는지 확인, 장보는 중에도 실시간으로 담은 물품의 합계 금액 확인 |

### 2.5 사용자 검증

**MVP 검증** (2025.06 ~ 07):
- MVP 범위: 랜딩페이지 → 장바구니 생성페이지 → 장보기 페이지
- 검증 포인트: "장바구니를 만들어서 실제 장보기 과정에서 사용했을 때 편리함을 느끼는가?"

**MVP 피드백** (2025.06.10 ~ 06.30, 구글 폼, MVP 사용자 98명 대상):
- **92.8%** 가 긍정적이라고 평가 (매우 도움 57.1%, 도움 35.7%)
- 긍정적 부분: 자동계산 기능 만족도 높음, 조작하기 편한 UI 구성
- 부정적 부분:
  - A. 장보기에서 아직 담지 않은 품목을 스크롤을 내려 찾아야 해서 불편함
  - B. [장바구니 생성 → 장보기 페이지] 이동 과정이 번거로워 잘 안쓰게 됨
  - C. 장보기 과정에서 품목을 담고 빼는 과정이 번거로움
- 개선 사항:
  - A. 품목 정렬을 '담은 품목'은 하단, '담지않은 품목'은 상단으로 배치
  - B. 심플모드를 선택할 수 있어 [품목 추가 → 체크] 만으로 사용 가능
  - C. 담기 모달 형식을 리스트에서 바로 체크할 수 있도록 UI 개선

### 2.6 프로젝트 비전
> 오프라인 장보기 이용자들의 삶의 질을 향상시켜줄 수 있는 서비스

"장보기"라는 인식이 귀찮고 시간 낭비가 아니라 그 자체로 즐겁고 힐링을 느낄 수 있는 하나의 여가활동으로 자리 잡을 수 있도록 한다.

## 4. 현재 구현 상태 (데모 v0.1.0)

### 4.1 페이지 구조

```
/ (랜딩)
├── /createCart (장바구니 생성)
└── /shopping/[id] (장보기)
```

### 4.2 구현 완료 기능

#### 4.2.1 랜딩 페이지 (`/`)
- 서비스 소개 및 브랜드 설명
- 주요 기능 미리보기 (스마트 장보기, 테마별 장바구니, 지출 관리)
- 서비스 시작 버튼

#### 4.2.2 장바구니 생성 페이지 (`/createCart`)
- 장바구니 이름 입력 (최대 20자)
- 품목 추가/삭제 (품명 + 수량)
- 메모 입력 (최대 100자, 장볼 때 참고 사항)
- 입력 데이터 localStorage 자동 저장
- 이전 장보기 세션 이어하기 안내 모달
- 장바구니 생성 가이드 모달

#### 4.2.3 장보기 페이지 (`/shopping/[id]`)
- **진행률 표시**: 전체 품목 대비 카트에 담은 비율 (프로그레스 바)
- **총 금액 합산**: 카트에 담은 물품의 예상 결제금액 실시간 표시
- **품목 담기/빼기**: 항목 클릭 시 수량/가격 입력 후 카트에 담기
- **자동 정렬**: 미구매 품목 → 구매 품목 순으로 자동 정렬
- **품목 추가**: 장보기 중 품목 추가 가능
- **메모 확인**: 플로팅 버튼으로 메모 내용 확인
- **장보기 완료**: 미구매 품목 확인 후 종료 (데이터 초기화)
- **사용 가이드**: 최초 방문 시 사용법 안내 모달
- **스티키 헤더**: 스크롤 시 진행률 + 금액 섹션 상단 고정

### 4.3 데이터 모델

```typescript
type CartItemType = {
  id: number;
  name: string;        // 품명
  quantity: number;     // 수량
  price?: number;       // 단가 (장보기 시 입력)
  status: 'IN_CART' | 'IN_LIST';  // 카트 담김 여부
};
```

**localStorage 키:**
| 키 | 용도 |
|----|------|

| `shoppingHistory` | 진행 중인 장보기 상태 |
| `shoppingUsageGuideShown` | 사용 가이드 노출 여부 |

## 9. 서비스 출시 후 기대효과

**사용자 측면:**
- All-In-One 서비스를 통한 장보기 정보의 분산 방지 (메모장, 계산기, 가계부를 한 곳에서 관리)
- 체계적인 지출 관리 (장보기 전·중·후 지출 모니터링으로 과소비 방지)
- 장보기의 힐링포인트 (레시피, 테마별 추천 장바구니 등으로 장보기의 재미 제공)

**마트 측면:**
- 오프라인 마트 방문률 증가 (서비스 이용이 늘수록 마트 방문 빈도 증가하는 선순환)
- 마트 홍보 창구의 역할 (오프라인 마트 이용자 유입을 통한 홍보효과 극대화)
