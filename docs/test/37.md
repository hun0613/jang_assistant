# #37 TC - 프론트엔드 localStorage → Supabase 마이그레이션

## 테스트 목적
localStorage 기반 데이터 로직을 Supabase Server Actions 기반으로 전환한 후,
기능 동작과 네트워크 요청 패턴이 올바른지 검증한다.

## 사전 조건
- `.env.local`에 Supabase 환경변수 설정 완료
- `pnpm dev` 실행 중
- 브라우저 개발자 도구 Network 탭 열기

---

## TC-37-01. `/createCart` 진입 (신규 사용자)

| 항목 | 내용 |
|------|------|
| 시나리오 | 장바구니 정보가 없는 신규 사용자가 진입 |
| 전제조건 | localStorage 초기화 (draftCartId, shoppingCartId, sessionId 모두 없음) |

**절차**
1. 브라우저 localStorage 완전 초기화
2. `/createCart` 진입

**기대 결과**
- [ ] 네트워크 탭에 POST 요청 **0건** (session/cart 자동 생성 없음)
- [ ] 폼이 빈 상태로 렌더링 (title, memo, items 모두 비어있음)
- [ ] Resume 모달 표시되지 않음

---

## TC-37-02. `/createCart` 진입 (draft 복원)

| 항목 | 내용 |
|------|------|
| 시나리오 | 작성 중이던 draft cart가 있는 경우 |
| 전제조건 | localStorage에 `draftCartId` 존재, 해당 cart가 DB에 CREATED 상태로 존재 |

**절차**
1. 이전 세션에서 제목/품목 입력 후 페이지 이탈
2. `/createCart` 재진입

**기대 결과**
- [ ] `getCartById` GET 요청 1건 발생
- [ ] `getCartItemsByCartId` GET 요청 1건 발생
- [ ] 이전에 입력한 title/memo/items가 폼에 복원됨

---

## TC-37-03. 최초 입력 시 cart 생성 (lazy creation)

| 항목 | 내용 |
|------|------|
| 시나리오 | 신규 사용자가 제목을 처음 입력 |
| 전제조건 | TC-37-01 상태 |

**절차**
1. 제목 입력란에 "테스트 장바구니" 입력
2. blur (다른 곳 클릭)

**기대 결과**
- [ ] POST 요청 2건: `createSession`, `createCart` (title 포함)
- [ ] localStorage에 `sessionId`, `draftCartId` 저장됨
- [ ] 추가 blur 시에는 POST 없이 `updateCart`만 호출

## TC-37-04. 품목 추가

| 항목 | 내용 |
|------|------|
| 시나리오 | 품목 추가 버튼 → 입력 → 추가 |

**절차**
1. "+ 품목 추가" 버튼 클릭
2. 품명 "사과", 수량 3 입력 후 "추가" 클릭

**기대 결과**
- [ ] "+ 품목 추가" 버튼 클릭 시 POST **없음** (모달만 열림)
- [ ] "추가" 클릭 시 `createCartItem` POST 1건
- [ ] 품목 리스트에 즉시 반영
- [ ] 모달 닫힘, 폼 초기화

## TC-37-07. 장보기 시작

**절차**
1. 제목 + 품목 1개 이상 입력
2. "장보기 시작하기" 클릭 → StartShoppingGuideModal
3. "시작" 클릭 (또는 5초 카운트다운)

**기대 결과**
- [ ] `updateCart` PATCH (status: SHOPPING) 1건
- [ ] localStorage에서 `draftCartId` 제거
- [ ] localStorage에 `shoppingCartId` 저장
- [ ] `/shopping/{cartId}`로 이동 (URL에 숫자 ID)

## TC-37-15. 동시 요청 방지 (Race condition)

**시나리오**: 여러 handler가 동시에 cart 생성 시도

**절차**
1. 제목 입력 후 빠르게 blur하면서 즉시 "+ 품목 추가" 클릭
2. 모달에서 바로 추가

**기대 결과**
- [ ] `createCart` 호출 **1건**만 발생 (중복 생성 없음)
- [ ] 두 handler가 같은 cartId를 공유
